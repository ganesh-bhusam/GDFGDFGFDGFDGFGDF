Here is exactly how the game separates players by language and handles creating new rooms when old ones get full. It's built on a very clean and efficient filtering system in `backend/gameEngine.js`:

When a player clicks **Play** and sends their chosen language (e.g., `langId = 0` for English) to the server, the server runs a function called `engine.publicRoomForLang(langId)`.

Here is the step-by-step logical flow of what happens:

### 1. The Search Phase
The server looks at a giant list of **all active rooms** currently running on the server and checks them one by one. For each room, it asks four strict questions:

1. **Is this a public room?** 
   *(It skips any private rooms created by friends).*
2. **Does this room's language match the player's language?** 
   *(If the room is Spanish `langId = 2` but the player wants English `langId = 0`, it skips the room entirely. This is how the strict language separation works).*
3. **Does the room have empty seats?** 
   *(It checks if `players.length` is less than the max players limit, which is 8. If the room already has 8 players, it skips it).*
4. **Is the user banned/kicked from this room?** 
   *(If the player was kicked out of this specific room earlier, it skips it).*

### 2. Allocation or Creation Phase
- **Match Found:** If a room passes **all** the checks above, the server assigns the player to that room. They instantly join the other players playing in that language.
- **No Match Found (Full or Empty):** If the server checks every single room and finds nothing (meaning all English rooms have 8/8 players, or there are zero English rooms currently active), the search function returns `null`.

### 3. The Fallback (Creating a New Room)
In `backend/server.js`, the code that handles this looks like this:
```javascript
room = engine.publicRoomForLang(langId) || engine.createPublic(langId);
```
Because the search returned `null`, the `||` (OR) statement kicks in and instantly triggers `engine.createPublic(langId)`. 

This creates a **brand new public room** bound to that exact language. The player is placed inside as the first player (1/8), and the room becomes available in the search pool. The next person who clicks "Play" in that language will be matched into this newly created room until it also fills up to 8 players.

---
This exact same logic is what we will use for the "Movies" mode! We just add a 5th strict question to the search phase: *"Does this room's mode match the player's mode?"* If not, it skips the room and eventually creates a new one for that mode.
